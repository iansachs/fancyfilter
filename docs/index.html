iOS Coding Exercise: Fancy Filter

Build a simple app using the Fancy Filter C++ library.

Goal

The goal is write a simple iOS app that allows the user to select a photo from the camera roll (or take a picture), and process it with the Fancy Filter C++ library. Consideration should be given to:

  • The interface between the iOS and C++ layers with an eye toward writing a clean and efficient wrapper for the Fancy Filter library
  • The design and user interaction.

We include an Xcode starter project in the repo to get you going. This exercise is less about the ability to write extremely complex iOS code, and more about API design, iOS best practices, and the ability to work, at least superficially, with C/C++ code.

Getting Started

  • Take a look at the Fancy Filter repo
  • Clone or download the latest Fancy Filter release. The important components are:
    • README.md
    • fancyfilter.hpp: C++ API definitions (fancyfilter::applyFilter in particular)
    • libff.a: FAT static library built for both the iphoneos and iphonesimulator platforms
    • fancyfilter-app/fancyfilter-app.xcodeproj: (optional) Starter Xcode project
    • fancyfilter-app/UIImage+Image3u.h/UIImage+Image3u.mm: (optional) UIImage category to simplify translating to/from Image3u

Rules

  • You can use any available resource you can find on the Internet — eg, Stack Overflow, API docs, etc.
  • You may not ask questions related to the problem on Stack Overflow, IRC, etc. (eg, the Internet is "read-only")
  • You cannot use any 3rd party libraries or products, you can only use classes and API’s in the standard iOS/Objective-C runtime. If in doubt, just ask.
